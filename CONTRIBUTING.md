# Contributing

Thank you for your interest in Addon Manager +.

## Before You Start

Open an issue before beginning significant work. This avoids duplicate effort and lets us align on approach before code is written.

## Guidelines

- **Keep changes focused.** One feature or fix per pull request.
- **Test with ExpressionEngine 7.** The target platform is EE 7; verify your changes in a real EE install.
- **Include screenshots for UI changes.** Add before/after screenshots to your pull request description.
- **Do not weaken ZIP or path safety.** Any change that relaxes path validation, allows `..` traversal, or removes rejection of unsafe ZIP entries will not be merged.

## Reporting Issues

Use GitHub Issues to report bugs or request features. Include:

- ExpressionEngine version
- PHP version
- Steps to reproduce
- Expected vs. actual behavior
